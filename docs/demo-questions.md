# Project A 演示问题模板

答辩前建议准备 10 到 20 份医疗规范、门诊指引或用药提醒文档，并围绕下面问题做演示。

如果你暂时没有自己的医疗文档，可以先使用仓库中的 `sample_docs/` 三份样例文件上传建库。

## 推荐演示问题

1. 高血压患者病情稳定后多久至少随访一次？
2. 高血压患者连续两次血压达到什么水平需要进一步评估？
3. 发热门诊分诊时需要记录哪些信息？
4. 哪些患者需要尽快安排医师评估？
5. 发热门诊什么情况下应立即启动绿色通道？
6. 抗菌药物是否可以作为普通病毒性感冒的常规治疗方案？
7. 使用青霉素类药物前需要先核对什么信息？
8. 首次静脉使用抗菌药物后要观察哪些异常反应？
9. 如果资料里没有某种疾病的治疗方案，系统应如何回答？
10. 如果用户要求直接给出处方结论，系统应该如何表现？

## 演示顺序建议

1. 先问一个文档中有明确条款的问题
2. 再问一个需要综合多句信息的问题
3. 最后问一个知识库中没有答案的问题，展示系统拒答能力

## 评分亮点

- 文档上传后能自动建库
- 问答结果有引用来源
- 没有依据时不会编造答案
- 后台能查看问答日志
